使用 Claude Code 的完整流程

发表于2025-10-12|更新于2025-10-12|工具心得

什么是 Claude Code？

Claude Code 是 Anthropic 推出的终端 AI 编程助手，直接集成在命令行中。不同于普通的代码补全工具，它能理解整个项目的架构和意图，可以读文件、写代码、执行命令、管理 Git，像一名真正的开发者一样与你协作。

VS Code 集成

Claude Code 也提供了 VS Code 扩展，可以在编辑器中直接使用，无需切换到终端。

核心命令

基础命令

命令 说明
/help 查看帮助文档
/clear 清除对话历史
/cost 查看当前会话消耗
/config 打开设置面板

工作流命令

命令 说明
!command 执行 shell 命令
/plan 生成实现计划
/review 审查代码变更
/fix 自动修复问题

推荐工作流程

1. 需求分析

在开始编码前，向 Claude Code 描述你的需求。上下文越详细，输出质量越高：

2. 规划阶段

使用 /plan 命令让 Claude Code 生成实现计划。它会分析项目结构，评估影响范围，然后给出分步实施方案。

3. 编码实现

Claude Code 会根据计划逐步实现功能。你可以让它在独立分支上工作，也可以通过 hexo 等隔离模式安全地进行大规模修改。

4. 代码审查

使用内置的代码审查智能体检查代码质量和安全性：

Claude Code 会自动检查：

  • 安全性：注入、XSS、认证漏洞等
  • 代码质量：复杂度、重复代码、命名规范
  • 性能：N+1 查询、缓存策略
  • 测试覆盖：是否缺少必要的测试

5. 提交代码

审查通过后，让 Claude Code 协助提交：

它会自动生成规范的提交信息。

智能体系统

Claude Code 提供了多种专业智能体，适合不同场景：

智能体 用途 使用时机
planner 实现规划 复杂需求开始前
code-reviewer 代码审查 每次修改代码后
tdd-guide 测试驱动开发 新功能开发时
security-reviewer 安全审计 涉及敏感操作时
build-error-resolver 构建错误修复 构建失败时
architect 架构设计 重大决策前

MCP 扩展

MCP（Model Context Protocol）允许 Claude Code 连接外部工具：

常用 MCP 服务：

  • GitHub：管理 Issue、PR、代码搜索
  • Playwright：浏览器自动化测试
  • Exa：网络搜索
  • Context7：文档查询

Hooks 系统

Hooks 允许在特定事件触发时自动执行脚本：

三种 Hook 类型：

  • PreToolUse：工具执行前，用于验证和拦截
  • PostToolUse：工具执行后，用于自动格式化和检查
  • Stop：会话结束时，用于清理和总结

最佳实践

1. 上下文越清晰越好

2. 分步推进

复杂功能分步骤完成，每步确认后再继续，避免一次性提出过多需求。

3. 善用并行智能体

独立任务可以同时处理，大幅提升效率：

4. 利用隔离模式

对于大规模修改，使用 hexo 隔离模式让 Claude Code 在独立副本中工作，不影响主分支。

总结

Claude Code 不仅仅是一个代码生成器，它是一个完整的 AI 编程助手。合理利用智能体系统、MCP 扩展和 Hooks 机制，可以显著提升开发效率。关键在于提供清晰的项目上下文和需求描述。
